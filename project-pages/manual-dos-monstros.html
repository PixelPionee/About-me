Manual de Campo do Caçador

Bestiário digital imersivo baseado na lore literária de The Witcher

Conceito

Este projeto é um manual de campo científico/naturalista que documenta criaturas sobrenaturais com:

  • ✅ Descrições fiéis aos livros de Andrzej Sapkowski
  • ✅ Mitologias originais (Eslava, Persa, Celta) quando o livro não detalha
  • ✅ Biologia realista e plausível
  • ✅ Estética de manual de campo envelhecido

⚠️ REGRA DE OURO: Ignorar completamente a estética dos jogos da CD Projekt RED

Este bestiário segue APENAS as descrições literárias, não as interpretações visuais dos jogos.

O Problema

Como criar um bestiário que seja fiel à lore literária (não aos jogos), com biologia realista e estética de manual de campo científico?

Bestiários de The Witcher frequentemente sofrem de problemas comuns:

  • Dependência excessiva da estética dos jogos (não dos livros)
  • Falta de fidelidade às descrições literárias de Sapkowski
  • Ausência de contexto mitológico real (Eslavo, Persa, Celta)
  • Biologia implausível ou inconsistente

O desafio era criar uma solução que priorizasse a lore literária original, mantendo rigor científico e estética de manual naturalista.

A Decisão Principal

SSG com Next.js 14 + Dados em JSON Local + Estética de Manual Envelhecido

Cada criatura é uma página estática gerada em build time, com dados estruturados em JSON que separam completamente conteúdo da apresentação. Isso garante:

  • 100% do conteúdo indexável por motores de busca
  • Performance máxima (páginas pré-renderizadas)
  • Fidelidade literária (dados baseados nos livros de Sapkowski)
  • Facilidade de manutenção (editar JSON é mais simples que editar HTML)

Stack Técnica

Next.js 14+

App Router com SSG nativo

TypeScript

Type safety para dados estruturados

CSS Modules

Estilização sem Tailwind

JSON Local

Sem banco de dados

Estrutura de Dados

Todas as criaturas estão em data/bestiario.json com a seguinte estrutura: 

interface Criatura {
  id: string;
  nome_comum: string;
  classificacao: string;
  ilustracao_src: string;
  habitat: string;
  comportamento: string;
  nivel_ameaca: string;
  observacoes_cacador: string;
}

Métricas de Qualidade

  • 100% conteúdo estático indexável por motores de busca
  • Fidelidade literária às descrições de Sapkowski
  • Separação total dados/interface para manutenibilidade
  • Type safety com TypeScript em toda a aplicação

O Que Foi Cortado (E Por Quê)

Decisões de escopo focadas em fidelidade literária e qualidade de leitura:

Sistema de Busca Client-Side

Motivo: Adicionaria complexidade prematura. Com poucas criaturas inicialmente, navegação por categorias é suficiente. Busca pode ser adicionada quando o conteúdo crescer.

Animações de Entrada

Motivo: Distração da leitura. O foco é conteúdo naturalista, não efeitos visuais. Animações sutis podem ser adicionadas depois, mas nunca devem competir com o texto.

Modo Escuro

Motivo: Foco em tema único bem executado (papel envelhecido). Melhor ter um tema que remete a manual antigo que dois temas medianos.

Sistema de Comentários

Motivo: Escopo e manutenção. Adicionar interatividade social exigiria backend, moderação e complexidade de infraestrutura desproporcional ao valor inicial.

Aprendizados

  • SSG resolve 80% dos problemas de performance e SEO: Páginas estáticas são rápidas, indexáveis e fáceis de hospedar. A complexidade de SPA só se justifica quando há necessidade real de interatividade dinâmica.
  • Separar dados da UI facilita manutenção: Editar conteúdo em JSON é mais simples e menos propenso a erros que editar diretamente em componentes. Também permite validação automática de estrutura.
  • Fidelidade literária > interpretações visuais populares: Seguir os livros de Sapkowski, não os jogos, cria um produto único e autêntico.
  • Conteúdo primeiro, features depois: É melhor ter conteúdo de qualidade em uma interface simples que uma interface complexa sem conteúdo. O valor está na informação, não na tecnologia.

Próximos Passos

Roadmap de desenvolvimento:

  1. Completar conteúdo inicial baseado nos livros de Sapkowski
  2. Implementar sistema de categorias por mitologia (Eslava, Persa, Celta)
  3. Adicionar ilustrações naturalistas (não baseadas nos jogos)
  4. Otimizar imagens e implementar lazy loading avançado
  5. Considerar busca client-side quando conteúdo atingir massa crítica

Nota: A estrutura do JSON está finalizada e não está sujeita a alterações. O conteúdo textual será refinado conforme a pesquisa literária avança.
